Methodology

Transparency over fake precision

This site uses a shaped sprint-estimation model with clear limits. It is designed to be helpful and honest, not official.

What the model is trying to do

The goal is to create a reasonable training-reference estimate from limited sprint data while still showing where the estimate becomes less trustworthy.

Sprinting is not linear

A 10-yard split is heavily about acceleration and early force application. A 100m race includes acceleration, upright max velocity, and speed maintenance. That is why the site uses shaped reference curves instead of pure distance-to-time multiplication.

Short sprints are acceleration-heavy

When you enter a short sprint, the model gives more weight to acceleration. If you add a 10-yard split, 20-yard split, or a 30m block time, the acceleration score becomes more informed.

Longer sprints involve max velocity and speed endurance

For 60m, 100m, and 200m outputs, the model gradually shifts more weight toward upright speed and the ability to hold speed over distance.

Timing method changes results

Hand timing, FAT, laser timing, and phone-video estimates do not create the same kind of number. The confidence rating drops when the timing setup is less reliable.

Start type changes results

Standing, 3-point, and block starts are not equivalent to rolling or flying starts. A rolling or flying input can still be useful, but standing-start comparisons become broader and less certain.

Hand time is not the same as FAT

The optional hand-time adjustment uses a +0.24 second track-style estimate. That is included as an educational reference only, and it should not be treated as an official conversion for every testing setup.

Benchmarks

Broad ranges only when the source is worth showing

No exact percentiles Benchmarks are broad general ranges, not exact percentiles or guarantees about level, recruiting, or future performance.
No unsupported tables If we do not have a trustworthy source for the exact sport/profile/distance combination, the site shows no benchmark instead of fabricating one.
Source notes stay visible Every benchmark used in v1 includes a source name, source link, notes, and a confidence label such as verified or general.

Confidence rating logic

How the site explains uncertainty

Low confidence

Usually one input only, hand or phone timing, no splits, no top speed, or a rolling/flying start being compared to a standing-start context.

Medium confidence

Usually one decent FAT or laser time with a reasonable start type and enough information to make a useful broad estimate.

High confidence

Usually FAT or laser timing with split data, top-speed context, and a clearly matched start type.
